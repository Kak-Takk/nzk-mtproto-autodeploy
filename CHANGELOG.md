# Журнал изменений

## [0.3.0] - 2026-04-05

### Добавлено
- **Telemt (Rust):** Новый движок на базе [telemt/telemt](https://github.com/telemt/telemt) — Rust/Tokio, без GC
- **Выбор движка:** При свежей установке — меню Telemt (рекомендуется) / MTG v2 (legacy)
- **Миграция:** При обновлении с MTG v2 — автоматический переход на Telemt с генерацией нового секрета
- **TCP Splicing:** Защита от Active Probing — сканер получает реальный сайт вместо дропа соединения
- **Middle-End Pool:** Быстрая загрузка фото/видео (решена проблема бесконечной загрузки медиа в mtg)
- **TLS-эмуляция:** Telemt копирует реальные TLS-записи целевого домена

### Изменено
- Секрет Telemt генерируется через `openssl rand -hex 16` (не требует Docker для генерации)
- Ссылки `tg://` автоматически формируются в формате `ee` + secret + hex(domain)
- Конфиг Telemt: `/root/.telemt/config.toml` (TOML вместо CLI-аргументов)
- Docker: убран `--read-only` для Telemt (нужен TLS-кэш), увеличен ulimit до 65536
- Вывод: баннер, статус и команды адаптированы под текущий движок

## [0.2.0] - 2026-03-23

### Добавлено
- **Модульная архитектура:** Исходный код разбит на 9 модулей в `lib/` для удобства поддержки
- **Сборочный пайплайн:** `scripts/build-single-file.sh` собирает `deploy_mt.sh` из модулей
- **Безопасный загрузчик конфига:** Парсер на основе белого списка вместо `source`
- **Hardening контейнера:** `--read-only`, `no-new-privileges`, лимиты памяти/PIDs, `tmpfs /tmp`
- **Stealth-профиль:** MSS=850 clamping + BBR (удалён опасный `tc netem` с RTT-отпечатком)
- **SNI-маршрутизация:** Nginx stream `ssl_preread` мультиплексирует сайты + MTProto на порту 443
- **Smoke-тест:** `smoke-mtproto.sh` — 23 автоматические проверки (Docker, TLS, iptables, трафик)
- **CI:** `.github/workflows/shell-ci.yml` для `bash -n` + `shellcheck`
- **Скрипт ротации:** `/root/rotate_fallback.sh` генерируется на сервере для ручной смены домена

### Изменено
- FakeTLS домены автоматически выбираются из RF-списка (yandex.ru, mail.ru и др.)
- Fallback на внешний сайт (microsoft.com) для неизвестных SNI вместо разрыва соединения
- Конфиги Nginx автоматически патчатся с `proxy_protocol` + восстановление реальных IP
