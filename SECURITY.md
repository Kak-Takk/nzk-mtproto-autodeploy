# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка |
|--------|-----------|
| 0.2.x  | ✅ Активная |
| < 0.2  | ❌ Нет      |

## Сообщение об уязвимости

**НЕ создавайте публичный Issue для уязвимостей безопасности.**

Используйте:

- **GitHub Security Advisories:** [Создать приватный отчёт](https://github.com/Kak-Takk/nzk-mtproto-autodeploy/security/advisories/new)

### Что считается уязвимостью

- Утечка секрета или конфига (через логи, файлы с открытыми правами и т.д.)
- Побег из контейнера или повышение привилегий
- Ошибка конфигурации Nginx, открывающая бэкенд-сервисы
- Векторы корреляции или деанонимизации трафика
- Выполнение произвольных команд через инъекцию в конфиг

### Что НЕ является уязвимостью

- Возможное обнаружение трафика через DPI (ограничение подхода FakeTLS, не баг в коде)
- IP VPS не совпадает с IP FakeTLS-домена (задокументированный компромисс)
- Запросы новых функций и обычные баги — используйте GitHub Issues

### Время реакции

Мы стараемся подтвердить получение отчёта в течение 48 часов и предоставить исправление в течение 7 дней для подтверждённых уязвимостей.
